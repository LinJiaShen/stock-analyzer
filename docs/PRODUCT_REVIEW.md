# 產品專業度評估 — 投資小白三場景檢視

> 評估日期：2026-06-11
> 評估視角：一位完全不懂技術分析的新手，能否在本平台完成三件事？

## 場景一：「我想快速理解一檔股票」

**動線**：首頁搜尋 2330 → 個股頁

| 能力 | 現況 | 評價 |
|------|------|------|
| 第一眼判斷 | 綜合評分大字 + 偏多/偏空徽章 + 52週位置 bar | ✅ 3 秒內有方向感 |
| 為什麼是這個分數 | AI 分析卡：白話總結 + 利多/利空觀點 + 注意事項 | ✅ 核心亮點，小白最需要的就是這個 |
| 該怎麼操作 | 操作建議卡：進場區間/停損/目標/風報比 | ✅ 有具體數字 |
| 看懂指標 | RSI/MACD/KDJ/均線各附白話注解；K線形態附解釋 | ✅ 本次補上 |
| 隔夜美股影響 | 美股關聯卡（ADR + 產業指數） | ✅ 本次補上 |
| 消息面 | 近期新聞 + 每則 LLM 評分與理由 | ✅ |

**結論：可以。** 個股頁已形成「評分 → 為什麼 → 怎麼做」的完整敘事鏈。

**仍缺**：
- 評分歷史趨勢圖（模型 `score_history` 已建，worker 每日存檔未排程）— 小白需要知道「分數是在變好還是變壞」
- 同業比較表（「台積電 44 分，那聯電幾分？」）— `IndustryService` 有 peers 邏輯，缺前端
- 基本面深度不足：目前「基本面分數」實際是產業相對強弱，**沒有 EPS/本益比/殖利率/營收年增**。對長線投資人這是最大缺口（資料源：TWSE OpenAPI `t187ap14_L` 月營收、`BWIBBU_d` 本益比/殖利率，皆免費）

## 場景二：「我想檢視自己的持股」

**動線**：登入 → 持倉管理 / 持倉分析

| 能力 | 現況 | 評價 |
|------|------|------|
| 損益一目了然 | 持倉管理有成本/現價/未實現損益 | ✅ |
| 健康檢查 | 持倉健診端點（評分+訊號+摘要） | ✅ 後端完成 |
| 風險提醒 | 停損價提示（操作建議卡） | ⚠️ 持倉頁未整合，要點進個股才看得到 |

**仍缺**：
- 投組層級視角：總曝險、產業集中度（「你 80% 都壓在半導體」）、與大盤相關性
- 持股異動推播：跌破停損/外資轉賣時主動提醒（需通知機制）

## 場景三：「我想找下一檔標的」

**動線**：篩選器 / 決策中心推薦 / 模擬交易驗證

| 能力 | 現況 | 評價 |
|------|------|------|
| 條件選股 | 8 種波動度/突破策略預設 + 自訂條件 | ✅ Graphcue 式 |
| 想法驗證 | 模擬交易（TP/SL/勝率/EV） | ✅ 差異化亮點 |
| 跟著訊號走 | 決策訊號（strong/watch/sell） | ⚠️ 訊號觸發門檻偏嚴，多數股票無訊號 |

**仍缺**：
- 篩選結果只有 52 檔有完整 K 線（全市場 2,373 檔已有清單與籌碼，K 線待回補）
- 「每日精選」首頁化：`/api/decision/recommendations` 已存在但沒有頁面入口

## 專業度總評

**架構面（後端）：偏專業水準。** 多因子加權模型、ATR 風控價位、OCO 模擬單、LLM 結構化輸出 + 規則式降級、TimescaleDB 時序庫 — 設計思路與商用產品（如 Graphcue）同路數。

**資料面：剛及格。** K 線僅 52 檔（全市場回補需排程跑數小時）、籌碼 22 日（建議補到 90 日讓「外資連買」更可靠）、基本面數據缺席。

**呈現面：本次大幅改善。** 紅漲綠跌統一（之前混用是專業性硬傷）、白話注解、新手三步驟動線。

### 優先級建議（下一輪）

1. **基本面數據**（高）：月營收年增 + 本益比 + 殖利率，補上評分的最後一塊拼圖
2. **全市場 K 線回補排程**（高）：篩選器才有實用價值
3. **評分歷史排程 + 趨勢圖**（中）：score_history 已有模型
4. **持倉頁整合健診與停損提示**（中）
5. **每日精選頁**（低）：recommendations 端點已在
